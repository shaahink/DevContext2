## What was done

### PLAN-8 — Desktop live-update gating

**Problem:** Every keystroke in the focus field (bound to `oninput`) cancelled and restarted the full analysis pipeline. Same for scenario, NoRoslyn, DryRun, and AntiPatterns toggles.

**Solution:** Split inputs into two tiers. Render-tier inputs (format, sections, max tokens, provenance, diagnostics) still auto-update instantly. Analysis-tier inputs (focus, scenario, no-roslyn, dry-run, anti-patterns) no longer auto-run — they either serve a cached snapshot from an in-memory LRU or mark the output stale with a banner + explicit Re-analyze button.

- `SnapshotCache` — session-scoped LRU (capacity 8) keyed by `AnalysisKey`
- `AnalysisKey` — identity record for analysis runs (ProjectPath, Scenario, Focus, Profile, NoRoslyn, DryRun, IncludeAntiPatterns). Render params excluded.
- `MarkAnalysisInputsChanged()` — cache-probe-or-stale, never auto-runs pipeline
- `AnalyzeAsync` — cache-hit fast path skips clone + pipeline entirely
- `IsStale` — computed property driving the amber stale banner
- Focus field switched from `@bind:event="oninput"` to `@bind` (fires on blur/Enter)

### Task 1 — Wire section toggles to filter output (P1)

**Problem:** The OutputPanel Sections drawer toggled `SectionViewModel.IsIncluded` (cosmetic token tally only, System B). The actual filter `SectionToggle.IsEnabled` (System A, drives `GetActiveSections()`) was only settable from `ConfigPanel.razor` — which was empty.

**Fix:**
- Added `Key` to `SectionViewModel`, populated from `SectionStat.Name` via reverse label→key mapping in `BuildSectionDataFromStat`
- Added `OnSectionToggled(string key, bool included)` callback to `SectionSelectionModel`
- Wired VM to call `SetSectionEnabled` via the callback
- Added `IsBuildingSections` flag to prevent re-entrant renders during section construction
- Added section checkboxes in `ConfigPanel.razor` under the previously-empty "Sections" block

### Task 2 — Profile in analysis identity (P2)

**Problem:** `DerivedProfile` (source→full, callgraph→debug, else→focused) is an analyze-time input but was absent from `AnalysisKey`. Section toggles that change profile (CallGraph, `__source__`) were classified as render-tier.

**Fix:**
- Added `string Profile` to `AnalysisKey`
- Included `DerivedProfile` in `BuildAnalysisKey()`
- Branched `SetSectionEnabled`: CallGraph/`__source__` → `MarkAnalysisInputsChanged()` (analysis-tier), others → `OnRenderInputChanged()` (render-tier)

### Task 3 — Multi-focus support (P1/P2)

**Problem:** `--focus`/`--around` are `string[]` (repeatable) but only `focusInput[0]` was used. `IntentInput.Focus` was a single `string?`.

**Fix:**
- Added `IReadOnlyList<string>? Focuses` to `IntentInput` (backward compat — `Focus` still works)
- Updated `AnalysisIntentResolver.Resolve` to loop over all focus texts
- Updated explanation string for multi-focus
- CLI passes full `focusInput` array as `Focuses`
- Desktop keeps single `Focus` (unchanged)

### Task 4 — Endpoint focus resolution (P2)

**Problem:** `FocusKind.Endpoint` was parsed but never resolved to its handler type. The explanation said "handler resolved after scan" but no such resolution existed.

**Fix:**
- Added `ResolveEndpointFocusPoints` method in `DiscoveryPipeline`, called after Stage 3 (`SpecificExtraction`) completes
- Matches `EndpointDetection` records by route + optional HTTP method
- Resolved endpoint focus points are rewritten to `FocusKind.Method` with the handler's `TypeName` and `MethodName`
- Unmatched routes emit a `DiagnosticLevel.Warning` diagnostic
- Updated `ResolveFocusPoints` failed-resolution filter to include `Endpoint` kind

### Task 5 — Clone cleanup modes (P2)

**Problem:** Only `auto` was implemented. `session`, `24h`, and `keep` were identical no-ops. The desktop defaulted to `24h` ("Cache 24h") but didn't implement it.

**Fix:**
- Added `IsCloneStale(path, maxAgeHours = 24)` — checks directory mtime age
- Added `RegisterForSessionCleanup(path)` and `CleanupSession()` using a static `ConcurrentBag<string>`
- Updated CLI cleanup to `switch` on all 4 modes
- Updated Desktop cleanup to `switch` on all 4 modes
- Desktop `Dispose()` calls `CleanupSession()` for session-mode clones

### Task 6 — Dead code removal (P3)

**Problem:** `TokenBudgetEnforcer` was unregistered in production DI but still existed as a source file. `TypeDiscovery.IsPruned` and `RelevanceScore` were never meaningfully set (enforcer was the only setter). 5 compression files carried dead `IsPruned` guards.

**Fix:**
- Deleted `TokenBudgetEnforcer.cs`
- Removed `IsPruned` and `RelevanceScore` from `TypeDiscovery`
- Simplified compression guards to `type.IsHardExcluded` only (6 files, 12 occurrences)
- Simplified `DiscoveryPipeline` before/after pruner counts to `model.Types.Values.Count`
- Removed enforcer registration from `EvalExpectationTests` and `GoldenTestHelper`
- Removed 2 TokenBudgetEnforcer tests from `PrunerTests`
- Updated `PipelineTests` to use `IsHardExcluded` instead of `IsPruned` for ordering test

### Task 7 — Polish (P3)

- **F9:** Guarded `AppendAntiPatterns` and `AppendEventFlow` with `ShouldRender` in `MarkdownRenderer`
- **F10:** Removed `--task` example from `Program.cs` (deprecation warning kept)
- **F11:** Changed `ConfigPanel.razor` ProjectPath binding to default `onchange` (debounce)
- **F12:** Removed dead `rootPath` parameter from `AddDevContextServices`, updated 3 call sites
- **Doc:** Added `html` to `--format` help text

## Key design decisions

1. **Cache key uses raw `ProjectPath`, not resolved clone dir** — enables cache probe before cloning a GitHub repo.
2. **SnapshotCache is not thread-safe** — called only from the UI thread. The VM is a DI singleton, so the cache is automatically session-scoped.
3. **`__source__` sentinel is never a real `SectionNames` entry** — left alone in all section-handling code.
4. **No CLI behavior or golden output was intentionally changed** — only Task 7 (F9) changes MarkdownRenderer output for Anti-patterns/EventFlow sections when filtered out.
5. **Golden expectation files (`-output.json`) are reference dumps** — not consumed by any test. Regeneration not needed.
